## what i believe

> design without code is a mood board. code without design is a mess. the interesting work lives in the middle.

- specific beats vague. "3 days fixing a sidebar" beats "worked on a project."
- ship before it's perfect. perfect isn't shipped.
- the craft is the constant. the domain changes.
- good UI is quiet. it doesn't ask you to notice it.
